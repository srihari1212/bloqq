Storage and Compute Optimization

Storage Optimization: There are four storage tokes such as 'Filecoin', 'Genaro', 'Siacoin' and 'Storj'. Each storage token is represented by the 9 variables e.g. 'Price', 'Fee', 'Blocktime', 'FCAS', 'Transection Volume', 'Exchange Volume', 'Bandwidth UP', 'Bandwidth Down' and Total storage'. The historical data against all nine variables and for all storage tokens are available. We used this data to train the machine learning models. Further the prediction against these variables are utilized to build the optimization model. The optimization model utilizes the preference variables and the prediction model to rank the storage tokens.

Compute Optimization: There are four compute tokes such as 'DCP', 'iExec(RLC)', 'Sonm(SNM)' and 'Golem(GLM)'. Each storage token is represented by the 8 variables e.g. 'Price', 'Blocktime', 'FCAS', 'Transection Volume', 'Exchange Volume', 'Deals', 'Instance Size' and 'GPU'. The historical data against all eight variables and for all storage tokens are available. We used this data to train the machine learning models. Further the prediction against these variables are utilized to build the optimization model. The optimization model utilizes the preference variables and the prediction model to rank the compute tokens.
